# 用户故事格式规范

## 标准格式

```
作为 [角色]
我想要 [功能/需求]
以便 [价值/收益]
```

## 示例

### 好示例
```
作为 用户
我想要 使用手机号登录
以便 快速访问我的个人账户

作为 管理员
我想要 查看用户列表
以便 管理用户权限
```

### 坏示例
```
登录功能                    # 缺少角色和价值
用户要能登录                # 格式不规范
弄个登录呗                  # 过于口语化
```

## 验收标准

好的用户故事应满足 INVEST 原则：

- **I**ndependent（独立的）
- **N**egotiable（可协商的）
- **V**aluable（有价值的）
- **E**stimable（可估算的）
- **S**mall（小的）
- **T**estable（可测试的）

## 检查清单

- [ ] 包含明确的角色
- [ ] 描述具体需求
- [ ] 说明业务价值
- [ ] 可独立开发
- [ ] 可测试验证
