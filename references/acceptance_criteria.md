# 验收标准编写指南

## 什么是好的验收标准

验收标准应该是：
- **具体**: 明确的条件和结果
- **可测试**: 可以验证是否满足
- **可量化**: 有明确的判断标准

## 示例

### 好示例
```
- [ ] 输入正确用户名和密码后可以成功登录
- [ ] 输入错误密码时显示"密码错误"提示
- [ ] 连续 5 次密码错误后锁定账户 30 分钟
- [ ] 登录页面加载时间 < 2 秒
```

### 坏示例
```
- [ ] 系统要好              # 模糊，不可测试
- [ ] 用户体验要好          # 主观，无法量化
- [ ] 性能要快              # 不具体
- [ ] 支持登录              # 不完整
```

## GIVEN-WHEN-THEN 格式

```
GIVEN [前置条件]
WHEN [操作]
THEN [预期结果]
```

### 示例
```
GIVEN 用户已注册
WHEN 用户输入正确的用户名和密码
THEN 用户成功登录并跳转到首页
```

## 检查清单

- [ ] 使用具体动词（可以/能够/显示/返回）
- [ ] 包含明确的输入和输出
- [ ] 可自动化测试
- [ ] 无歧义
- [ ] 至少 3 条验收标准
